Stapelverarbeitung

Der Callcenter Simulator kann über die Funktion Stapelverarbeitung mehrere Simulationsläufe in unmittelbarer Folge durchführen. Es werden zwei Arten von Stapelverarbeitungen unterstützt:

Verzeichnis abarbeiten

In diesem Modus muss lediglich ein Verzeichnis, in dem sich die zu simulierenden Modelle befinden, angegeben werden. Der Simulator durchsucht dann das angegeben Verzeichnis nach xml-Dateien, die Callcenter-Modelle enthalten und zu denen noch keine passenden Dateien mit dem Namen *_Statistik.xml existieren, simuliert die entsprechenden Modell und speichert die Ergebnisse unter den Namen *_Statistik.xml in demselben Verzeichnis.

Diese Funktion kann auch über Kommandozeilenparameter verwendet werden.

Parameterreihe erstellen

In diesem Modus wird das aktuell im Editor geladene Modell simuliert. Allerdings wird ein Modellparameter bei jedem Lauf verändert, so dass am Ende eine Reihe von Simulationen mit einem einen Bereich durchlaufenden Parameter vorliegen. Die Simulationsergebnisse werden in dem angegebenen Verzeichnis in Form von Statistik-xml-Dateien ausgegeben. Bitte beachten Sie hier zu auch, dass die Ergebnisse auch gemeinsam in aufbereiteter Form in eine einzige Datei geschrieben werden können (siehe hierzu der dialog "Weitere Ausgabeeinstellungen").

Dialog "Weitere Ausgabeeinstellungen"

Über diesen Dialog kann eingestellt werden, dass direkt im Anschluss an eine Simulation ein Ausgabefilter auf die Ergebnisse angewandt werden soll und die Ergebnisse an eine Datei angehängt werden sollen. Auf diese Weise kann insbesondere bei der Erstellung einer Parameterreihe eine Tabelle, die direkt die Ergebnisse enzhält, erstellt werden.
