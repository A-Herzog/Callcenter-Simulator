Kommandozeilenparameter

Bitte beachten Sie, dass nur die kommerzielle Version des Callcenter Simulators per Kommandozeilenparameter gesteuert werden kann.

Allgemein kann der Simulator auf drei Arten mit Parametern aufgerufen werden:

  1. Aufruf mit einer Modell-Datei
    <SimulatorProgrammdatei> <Simulationsmodell>
    Startet die grafische Oberfläche des Simulators und lädt die angegebene Modell-Datei.
  2. Aufruf mit einer Statistik-Datei
    <SimulatorProgrammdatei> <Statistikdaten>
    Startet die grafische Oberfläche des Simulators und lädt die angegebene Statistik-Datei.
  3. Aufruf mit einem Befehl
    <SimulatorProgrammdatei> <Befehl> <Parameter>
    Startet den Simulator im Kommandozeilen-Modus und führt den angegebenen Befehl aus.

Zusätzlich kann zwischen dem Namen der Programmdatei (<SimulatorProgrammdatei>) und den weiteren Parametern noch angegeben werden, wo der Simulator die Programmkonfiguration ablegen soll. Dies geschieht wie folgt:

<SimulatorProgrammdatei> cfg="<PfadUndDateiname>" <OptionalWeitereParameter>

Beim Programmstart wird zunächst die Konfigurationsdatei im Simulator-Programmverzeichnis geladen und dann mit der Benutzer-Konfigurationsdatei überlagert. Schreibzugriffe erfolgen ausschließlich auf die benutzerdefinierte Konfigurationsdatei.

Befehle für den Kommandozeilen-Modus

Achtung: Der Kommandozeilen-Modus steht nur zur Verfügung, wenn das Programm als jar-Datei ausgeführt wird. Die Windows-exe-Datei kann nicht über die folgenden Befehle gesteuert werden.

Der Platzhalter <Befehl> in der 3. Aufrufvariante die in den folgenden Abschnitten beschriebenen Inhalte annehmen.

MODELLAUSTABELLEN

Erstellt aus einer Anrufer- und einer Agententabelle ein Callcenter-Modell.
Der Befehl erwartet drei Parameter:
1. Kunden-Eingabe-Tabelle
2. Agenten-Eingabe-Tabelle
3. Dateiname unter dem das Modell gespeichert werden soll.
Die beiden Tabellennamen können, jeweils durch "!" abgetrennt um den Namen der Tabelle innerhalb der Arbeitsmappendatei und um die erste zu berücksichtigende Spalte ergänzt werden.
Die Anrufertabelle kann beliebig viele Spalten enthalten. Pro Spalte wird ein Kundentyp definiert. Jede Spalte muss aus folgenden Zeilen bestehen:

  • Name des Kundentyps
  • 48 Zeilen für die Anzahlen an Anrufen pro Halbstundenintervall
  • Basispriorität des Kundentyps
  • Prioritätsänderung pro Wartesekunde
  • Prioritätsänderung bei Weiterleitungen
  • Mittlere Wartezeittoleranz
  • Wiederholwahrscheinlichkeit
  • Mittlerer Wiederholabstand
  • Beliebig viele weitere Zeilen, die angeben mit welcher Wahrscheinlichkeit Kunden dieses Typs nach einem Gespräch weitergeleitet werden (Reihenfolge entsprechend der Spaltenreihenfolge der Tabelle; leere Zeilen bedeuten, dass Kunden nicht zu dem jeweiligen Typ weitergeleitet werden)

Die Agententabelle kann beliebig viele Spalten enthalten. Pro Spalte wird eine Agentengruppe definiert. Jede Spalte muss aus folgenden Zeilen bestehen:

  • Name der Gruppe bzw. des Skill-Levels
  • 48 Zeilen für die Agentenanzahlen pro Halbstundenintervall
  • Beliebig viele weitere Zeilen für die mittleren Bedienzeiten der Kundentypen (Reihenfolge entsprechend der Spaltenreihenfolge der Kundentabelle; leere Zeilen bedeuten, dass die Agentengruppe Kunden dieses Typs nicht bedienen kann)

XMLVORVERARBEITUNG

Fügt Zellen aus einer Tabelle in eine XML-Datei ein.
Der Befehl erwartet 3 Parameter:
1. Die Modell-Eingabe-Datei
2. Die Tabellendatei, aus der die Daten entnommen werden sollen
3. Die Modell-Ausgabe-Datei
In der Eingabe-Modelldatei könnten an beliebiger Stelle Ausdrücke der Form $(Tabellenname!A1) oder $(Tabellenname!A1:A48) stehen, die angegeben dass eine einzelne Zelle (im Beispiel A1) oder ein ganzer Bereich (im Beispiel die Zellen von A1 bis A48) an der angegebenen Stelle in die XML-Datei eingefügt werden sollen. Der Tabellenname ist dabei optional. Wird keine Tabelle angegeben, also z.B. $(A1) oder $(A1:A48) so wird die erste Tabelle der Arbeitsmappe verwendet.

VERLAGERN

Kunden oder Agenten verlagern um Ressourcen besser ausnutzen zu können.
Dieser Befehl erwartet vier or mehr weitere Parameter:
1. Eingabe-Modell-Datei
2. Ausgabe-Statstik-Datei
3. Veränderungsmodus
4. Veränderungsstärke
Die Eingabedatei muss existieren, die Ausgabedatei darf nicht existieren. Der Veränderungsmodus muss entweder "Kunden" oder "Agenten" lauten und die Veränderungsstärke eine Zahl zwischen 0 und 1 sein.
Wenn keine weiteren Parameter angegeben werden, wirken sich die Veränderungen auf alle Kunden- bzw. Agentengruppen aus. Wenn weitere Parameter angegeben werden, werden nur die Gruppen, die im 5., 6. usw. Parameter benannt sind, verändert.

VORPLANUNG

Erlang-C-basierte Vorplanung der Anzahl an Agenten.
Dieser Befehl erwartet genau vier weitere Parameter:
1. Dateiname des Ausgangsmodells
2. Dateiname des Ergebnismodells
3. Name der einzuhaltenen Kenngröße
4. Wert der einzuhaltenen Kenngröße
Der Name der Kenngröße kann dabei einer der folgenden Werte sein: "Erreichbarkeit", "Wartezeit" oder "ServiceLevel", wobei noch jeweils (ohne Leerzeichen) "Wiederholer" und/oder "Vereinfacht" an den Namen der Kenngröße angehängt werden können (in dieser Reihenfolge). Wird einer der "...Wiederholer"-Werte verwendet, so werden in dem Erlang-C-Modell Wiederholer berücksichtigt. Wird einer der "...Vereinfacht"-Werte verwendet, so wird das Modell vor der Berechnung der Agentenzahlen zu einem Single-Skill-Modell vereinfacht.

Vereinfacht das Modell zu einem Single-Skill-Modell.

1. Eingabe-Modell-Datei
2. Ausgabe-Modell-Datei

Der Befehl erwartet als Parameter eine nicht-negative ganze Zahl.
