Optimierer

Mit Hilfe des Optimierers kann ermittelt werden, wie viele Agenten notwendig sind, um bestimmte Service-Kenngrößen zu erreichen. Dabei kann sowohl eingestellt werden, welche Kenngröße über welche Kundengruppe erreicht werden soll, als auch, welche Agentengruppe verändert werden soll, um sich der Zielgröße anzunähern.

Als Ausgangsmodell wird das aktuelle Modell aus dem Callcenter-Model-Editor verwendet. Das Modell wird mit jedem Simulationslauf verändert, bis sich die gewünschten Werte einstellen. Dabei bleibt jedoch das Ursprungsmodell im Editor unverändert.

Nach Ende des Optimierungsprozesses werden die Ergebnisse im Optimierer-Ergebnisbetrachter angezeigt. Es werden dabei sowohl die Veränderungen über die Simulationsläufe hinweg dokumentiert als auch die Statistik-Ergebnisse der einzelnen Läufe gespeichert.

Bevor über die Optimierung starten Schaltfläche der Optimierungsvorgang gestartet werden kann, muss konfiguriert werden, wie die Optimierung ablaufen soll:

Optimierungsgröße

Zu optimierende Eigenschaft und deren Größe

Über dieses Auswahlfeld und die folgende Eingabezeile kann eingestellt werden, was optimiert werden soll und welcher Wert erreicht werden soll. Es stehen dabei sowohl Eigenschaften der Kunden (Erreichbarkeit, mittlere Wartezeit, mittlere Verweilzeit und Service-Level) als auch Eigenschaften der Agenten (Auslastung) zur Auswahl. Außerdem kann ein zweiter Wert eingestellt werden, bei dessen Überschreitung (Service-Level, Erreichbarkeit) bzw. Unterschreitung (Wartezeit, Verweilzeit, Auslastung) die Anzahl an Agenten reduziert wird. Wird diese Option nicht explizit aktiviert, so erfolgt nur eine Optimierung hin zu größeren Agentenzahlen.

Zielgröße muss erreicht werden über ... (Zeitraum)

In diesem Auswahlfeld lässt sich einstellen, ob die Zielgröße nur im Durchschnitt über den Tag oder aber in jedem einzelnen Halbstundenintervall erreicht werden muss.

  • Wird über den gesamten Tag optimiert, so kann es vorkommen, dass die Zielgröße in einzelnen Intervalle evtl. nicht erreicht wird, dies aber durch besonders gute Werte in Intervallen mit vielen Kunden bzw. vielen Agenten kompensiert wird.
  • Ist die Option In jedem einzelnen Intervall ausgewählt, so muss die Zielgröße in jedem einzelnen Intervall erreicht werden. Wird die Zielgröße in einem Intervall besonders stark über- bzw. unterschritten, so wird dies nicht auf Intervalle, in denen das Optimierungsziel noch nicht erreicht ist, angerechnet.
  • Die Option In jedem einzelnen Intervall (der Reihe nach optimieren) stellt eine Erweiterung der vorherigen Option dar. Die Zielgröße muss, wenn diese Option gewählt ist, nicht nur in jedem einzelnen Intervall erreicht werden, sondern der Optimierer arbeitet während des Optimierungsvorgang außerdem intervallweise. Dies bedeutet, dass, wenn die Zielgröße in zwei zeitlich aufeinander folgenden Intervallen unterschritten wird, die Agentenzahlen vor dem nächsten Simulationslauf nicht in beiden Intervallen angehoben werden, sondern zunächst nur in dem zeitlich früheren Intervall. Erst wenn in diesem die Zielgröße erreicht wurde, wird das nächste Intervall im Optimierer betrachtet. Diese Vorgehensweise führt häufig zu geringeren Agentenzahlen um dieselben Zielwerte zu erreichen.

    Wird die Zielgröße in zwei aufeinanderfolgenden Intervallen unterschritten und wird daher die Agentenzahl in beiden Intervallen erhöht, so verbessern sich die Service-Kenngrößen in beiden Intervalen. Dadurch entstehen auch weniger Abbrecher in dem ersten Intervall, von denen viele im zweiten Intervall als Wiederholer erneut aufgetreten wären. D.h. die Auslastung sinkt im zweiten betrachteten Intervall und evtl. wäre es daher dann überhaupt nicht nötig gewesen, die Agentenanzahl auch in diesem Intervall anzuheben. Genau dieser Effekt wird durch die intervallweise Optimierung ausgeschlossen.

Zielgröße muss erreicht werden über ... (Kunden- bzw. Agentengruppen)

In diesem Feld kann eingestellt werde, welche Kunden- bzw. Agentengruppen die Zielgröße erreichen sollen. Soll die Zielgröße über alle Gruppen erreicht werden, so kann außerdem eingestellt werden, ob sie von jeder Gruppe einzeln oder im Durchschnitt über alle Gruppen erreicht werden muss.

Stellgröße

Anzahl an Agenten in jedem Lauf erhöhen um Prozentwert

In diesem Eingabefeld kann eingestellt werden, um wie viel Prozent die Agentenanzahl in den betroffenen Intervallen erhöht werden soll, wenn die Zielgröße in einem Simulationslauf nicht erreicht wurde.

Ergab die Erhöhung der Zahlen gerundet auf ganze Agenten über den gesamten Tag überhaupt keine Veränderung, so wird nochmals derselbe Prozentwert auf die Agentenzahlen aufgeschlagen, bevor der nächste Simulationslauf gestartet wird.

Der Basiswert, der 100% entspricht ist dabei stets die Ausgangsagentenanzahl. D.h. 1% entsprechen stets demselben absoluten Zuwachs an Agenten.

Einschränkungen

Über diesen Dialog können pro Agentengruppe und pro Halbstundenintervall Minimal- und Maximalwerte definiert werden, die nicht unter- bzw. überschritten werden dürfen. Der Simulator versucht dann im Rahmen dieser äußeren Einschränkungen die gewünschten Ziele in der Optimierung zu erreichen.

Agentenzahl in folgenden Gruppen erhöhen

In diesem Auswahlfeld kann festgelegt werden, welche Agentengruppen vergrößert werden sollen, um die Zielgröße zu erreichen. Es können entweder alle oder nur einige Agentengruppen pro Simulationslauf verändert werden.

Intervalle

Auf dieser Seite kann eingestellt werden, welche Intervalle bei der Optimierung (in Bezug auf die Kenngrößen und in Bezug auf die Veränderung der Agentenzahlen) berücksichtigt werden sollen. Als Vorgabeeinstellung werden alle Intervalle berücksichtigt, d.h. die Optimierung erfolgt über den gesamten Tag.

Übertrag

Auf dieser Seite kann ein Übertrag von einem vorherigen Tag, der in das Simulationsmodell eingespeist werden soll, definiert werden. Über die Angabe einer Statistik-Datei von einem vorherigen Simulationslauf kann ein direkter Übertrag von einem Vortrag eingerichtet werden. Des Weiteren besteht die Option, manuell einen Anfangsbestand an Kunden, der zu Beginn der Simulation anwesend ist, zu definieren. Ein exakter Übertrag der einzelnen Kunden inkl. derer Restwartezeittoleranz usw. ist nur bei der Verwendung einer Statistik-Datei möglich.
