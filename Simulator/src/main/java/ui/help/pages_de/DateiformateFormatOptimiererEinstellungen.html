Das Basis-Element für Optimierer-Einstellungen-Dateien heißt <OptimiererEinstellungen>.

Angaben zur Optimierung

Jede Optimierer-Einstellungen-Datei muss von jedem der folgenden Elemente, sofern nicht anders angegeben, jeweils genau eins enthalten:

  • <Optimierung-Eigenschaft>: Dieses Element kann als Inhalt in Abhängigkeit davon, welche Größe optimiert werden soll, einen der folgenden Werte annehmen: Erreichbarkeit, Erreichbarkeit(Kundenbasis), Wartezeit, Wartezeit(Kundenbasis), Verweilzeit, Verweilzeit(Kundenbasis), ServiceLevel(Anrufbasis), ServiceLevel(Anrufbasis;Alle), ServiceLevel(Kundenbasis), ServiceLevel(Kundenbasis;Alle) oder Auslastung.
  • <Optimierung-Wert>: In Abhängigkeit von dem Wert in dem Element <Optimierung-Eigenschaft> muss in diesem Element eine Wahrscheinlichkeit oder eine Zeit angegeben werden.
  • <Optimierung-WertMaximum>: Über dieses optionale Element kann eine zweite Grenze für den zu optimierende Größe festgelegt werden. Bei der Optimierung wird die Agentenzahl zunächst so lange erhöhe, bis die Primärgrenze eingehalten wird. Danach wird ggf. die Agentenzahl wieder reduziert, um diese sekundäre Grenze einzuhalten.
  • <Optimierung-Intervalle>: Dieses Element kann als Inhalt einen der folgenden Werte annehmen:
    • Durchschnitt: Der Zielwert muss lediglich im Durchschnitt über den Tag erreicht werden. Dies bedeutet, dass allen Anrufern dasselbe Gewicht beigemessen wird.
    • Einzeln: Der Zielwert muss in jedem Halbstundenintervall erreicht werden. Dies bedeuet, dass auch in Intervallen mit wenigen Anrufern der Zielwert erreicht werden muss und folglich die Anrufer in derartigen Intervallen einen verhältnismäßig großen Einfluss auf den Optimierungsprozess haben.
    • Reihenfolge: Der Zielwert muss in jedem Halbstundenintervall erreicht werden. Darüberhinaus erfolgt die Optimierung intervallweise, d.h. die Agentenzahlen in einem Intervall werden erst dann angepasst, wenn diese in allen vorangegangenen Intervallen bereits erreicht wurde. Durch diese Methode verlängert sich die Zeit für den gesamten Optimierungsprozess erheblich, allerdings wird durch diese Methode auch ausgeschlossen, dass sich die Agentenzahl in einem Intervall nachträglich als zu hoch erweist, weil später noch in einem vorherigen Intervall weitere Agenten hinzugefügt wurden und somit der Übertrag an Kunden in das nächste Intervall verringert wurde.
  • <Optimierung-Gruppen>: Dieses Element kann als Inhalt einen der folgenden Werte annehmen:
    • Durchschnitt: Die Zielgröße muss im Durchschnitt über alle Kundengruppen bzw. Callcenter erreicht werden.
    • Alle: Die Zielgröße muss von jeder einzelnen Kundengruppen bzw. jedem Callcenter erreicht werden.
    • Auswahl: Die Zielgröße muss von denen per <Optimierung-Gruppe>-Elementen angegebenen Kundengruppe bzw. Callcentern erreicht werden.
  • <Anpassung-Wert>: Gibt den Prozentwert an, um wie viel die Agentenanzahl in jedem Optimierungsschritt in jedem anzupassenden Intervall angehoben werden soll.
  • <Anpassung-Gruppen>: Dieses Element kann als Inhalt einen der folgenden Werte annehmen:
    • Alle: Es werden in jedem Optimierungsschritt jeweils alle Agentengruppen in den entsprechende Intervallen angepasst.
    • Auswahl: Es werden in jedem Optimierungsschritt jeweils die in den <Anpassung-Gruppe>-Elementen angegebenen Agentengruppen in den entsprechende Intervallen angepasst.
  • <Optimierung-InitialerBestand>: Dieses Element ist optional. Das <Optimierung-InitialerBestand>-Element kann beliebig viele <Optimierung-InitialerBestandKundentyp>-Elemente beinhalten. Jedes <Optimierung-InitialerBestandKundentyp>-Element muss ein Name-Attribut besitzen, das angibt auf welchen Kundentyp sich das Element bezieht und muss als Inhalt eine nichtnegative ganze Zahl enthalten, die angibt, wie viele zusätzliche Erstanrufer als manueller Übertrag zu dem Simulationstag hinzugefügt werden sollen.
  • <Einschraenkung-Gruppe>: Dieses Element ist optional. Es können beliebig viele <Einschraenkung-Gruppe> existieren. Jedes muss die drei Attribute Name, Minimum und Maximum besitzen. Jedes <Einschraenkung-Gruppe>-Element beschreibt die Einschränkungen bei der Optimierung bzgl. der Agentenanzahlen in der über das Name-Attribut angegebenen Agentengruppe. Minimum und Maximum müssen jeweils 48 durch ";" getrennte Werte enthalten, die die Minimal- und Maximalanzahl an Agenten pro Halbstundenintervall angeben.

Optional kann in dem <OptimiererEinstellungen>-Element ein <Optimierung-Optimierungsintervalle>-Element enthalten sein. Wenn dieses Element existiert, so muss es 48 durch ";" getrennte Werte, die entweder "0" oder "1" sind, enthalten. Diese Werte geben an, welche Intervalle bei der Optimierung berücksichtigt werden sollen. Fehlt dieses Element, so werden alle Intervalle berücksichtigt.

Ist in dem <Optimierung-Gruppen>-Element der Inhalt Auswahl angegeben, so müssen Elemente des Typs <Optimierung-Gruppe> angelegt werden, die angeben, welche Kundengruppen bzw. Callcenter den Zielwert erreichen sollen. (Für jede zu berücksichtigende Kundengruppe bzw. jedes zu berücksichtigende Callcenter muss jeweils ein <Optimierung-Gruppe>-Element angelegt werden, welches als Inhalt den Namen einer Kundengruppe bzw. des Calcenters enthält.)

Ist in dem <Anpassung-Gruppen>-Element der Inhalt Auswahl angegeben, so müssen Elemente des Typs <Anpassung-Gruppe> angelegt werden, die angeben, welche Agentengruppen jeweils verändert werden sollen. (Für jede zu ändernde Agentengrtuppe muss jeweils ein <Anpassung-Gruppe>-Element angelegt werden, welches als Inhalt die Nummer der Agentengruppe innerhalb des Callcenters (1-basierend) gefolgt von einem Bindestrich und dem Namen des Callcenters enthält.)

Außerdem kann optional in dem <OptimiererEinstellungen>-Element ein <Optimierung-Uebertrag>-Element existieren. Dieses <Optimierung-Uebertrag>-Element kann optional das Attribut StatistikDatei besitzen, welches angibt, aus welcher Datei der Kundenübertrag vom Vortrag geladen werden soll. (Existiert das <Optimierung-Uebertrag>-Element nicht oder besitzt es kein StatistikDatei-Attribut, so erfolgt kein Kundenübertrag.) Desweiteren kann das <OptimiererEinstellungen>-Element beliebig viele <VerknuepftUebertrag>-Unterelemente besitzen. Siehe dazu die Dokumentation von <VerknuepftUebertrag>-Element bei verknüpften Simulationsmodellen.
