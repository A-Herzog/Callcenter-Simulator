Schnellzugriff

Der Ergebnis-Schnellzugriff ermöglicht es, Daten von ganzen Simulationsreihen immer wieder in derselben Form z.B. zwecks Speicherung in Tabellen zu exportieren. Über die Filtereinstellungen am unteren Rand der Seite kann dabei sowohl eingestellt werden, welche Daten exportiert werden sollen, als auch wie die Ausgabe formatiert werden soll.

Filterbefehle

In das Filter-Skript-Eingabefeld am unteren Rand der Seite können beliebig lange Skripte eingegeben werden. Ein Skript besteht dabei jeweils aus einer Anweisung pro Zeile. Das aktuelle Skript wird beim Beenden des Simulators automatisch gespeichert und beim nächsten Start wiederhergestellt.

Eine Filter-Skript-Anweisung besteht stets aus einem Befehl gefolgt von einem Leerzeichen und dann den Parametern des Befehls. Bei den Befehlen selbst wird nicht zwischen Groß- und Kleinschreibung unterschieden. Bei den Parametern der Befehle zur Textausgabe und zur Filterung der XML-Daten wird die jeweilige Schreibweise jedoch berücksichtigt. Leerzeilen werden bei der Verarbeitung ignoriert. Des Weiteren wird alles, was hinter einem "//" in einer Zeile steht als Kommentar interpretiert und nicht weiter verarbeitet.

Ein Filter-Skript kann aus den folgenden Arten von Befehlen bestehen:

Format

Über den Format-Befehl kann angegeben werden, ob die folgenden Zahlenwerte in lokaler Notation (im deutschsprachigen Raum mit einem Dezimalkomma) oder in System-Notation mit einem Dezimalpunkt ausgegeben werden sollen. Außerdem kann angegeben werden, ob Zahlenwerte als Prozentangabe ausgegeben werden sollen. In diesem Fall wird der Wert mit 100 multipliziert und ein "%"-Zeichen an die Zahl angefügt. Voreingestellt ist stets die Ausgabe in lokaler Notation und die Ausgabe als normale Fließkommazahl (also nicht als Prozentwert).

  • Wahl der System-Notation für Zahlen und Prozentwerte: Format System
  • Wahl der lokalen Notation für Zahlen und Prozentwerte: Format Local
  • Wahl der Ausgabe als normale Zahl: Format Fraction (z.B. 0,357 oder 0.375)
  • Wahl der Ausgabe als Prozentwert: Format Percent (z.B. 35,7% oder 35.7%)
  • Ausgabe der Zahlenwerte als Zeitangaben: Format Time (z.B. 00:03:25,87)
  • Ausgabe der Zahlenwerte als normale Zahlen: Format Number (Ausgabe gemäß Angabe Percent oder Fraction)

Separator

Über den Separator-Befehl kann angegeben werden, wie die einzelnen Werte bei der Ausgabe einer Verteilung getrennt werden sollen.

  • Semikolons als Trenner: Separator Semicolon
  • Zeilenumbrüche als Trenner: Separator Line
  • Tabulatoren als Trenner: Separator Tabs

Text

Über diesen Befehl kann ein beliebiger Text (z.B. Spaltentrenner) ausgegeben werden. Der Text selber muss in doppelten Anführungszeichen stehen. Als besondere Anweisungen innerhalb des Textes werden \t und \n unterstützt. \t fügt einen Tabulator ein und \n erzeugt einen Zeilenumbruch.

Beispiel: Text "Hallo Welt!\n"

Title

Über diesen Befehl kann ein beliebiger Titel für das Skript definiert werden. Dieser Titel wird in der Dropwdown-Box bei der Auswahl des Skriptes angezeigt. Der Titel selber muss in doppelten Anführungszeichen stehen.

Beispiel: Title "Zusammenfassung Wartezeiten"

Print

Über den Print-Befehl können beliebige Einträge aus dem XML-Statistikbaum, Variablen und die Ergebnisse einfacher Berechnungen ausgegeben werden.

Beispiele:

  • Print CallcenterModell->ModellName
    Liefert den Inhalt des Elements ModellName, welches ein Unterelement von CallcenterModell ist.
  • Print x
    Gibt den Wert der Variable x aus.
  • Print (x+3)*4
    Berechnet den Wert (x+3)*4 unter Berücksichtigung von Punkt- vor Strichrechnung und gibt das Ergebnis aus (siehe auch Rechenbefehle).

Ist keine Variable mit dem als Parameter angegebenen Namen gesetzt und kann der Parameter auch nicht aus Ausdruck ausgerechnet werden, so wird der Parameter als XML-Auswahlbefehl interpretiert.

Set

Speichert einen beliebigen Eintrag aus dem XML-Statistikbaum in einer Variable.

Beispiel: Set x=CallcenterModell->ModellName

Calc

Führt eine Berechnung mit Variablen durch und speichert das Ergebnis wieder in einer Variable (siehe auch Rechenbefehle).

Beispiel: Calc y=(x+3)*4

Mean

Liefert den Mittelwert einer als Parameter übergebenen Verteilung.

Beispiel: Mean CallcenterModell->ModellKundentyp[Name="KundenA"]->KundenWartezeittoleranzVerteilung

SD

Liefert die Standardabweichung (Standard Derivation) einer als Parameter übergebenen Verteilung.

Beispiel: SD CallcenterModell->ModellKundentyp[Name="KundenA"]->KundenWartezeittoleranzVerteilung

CV

Liefert den Variationskoeffizient (Coefficient of Variation) einer als Parameter übergebenen Verteilung.

Beispiel: CV CallcenterModell->ModellKundentyp[Name="KundenA"]->KundenWartezeittoleranzVerteilung

Sum

Liefert Summe über die als Parameter übergebenen empirischen Verteilung.

Beispiel: Sum StatistikKunden[Name="KundenA"]->StatistikKundenWartezeitverteilung[Verteilung]

XML-Auswahlbefehle

Die Befehle Print und Set können Daten aus dem XML-Baum auslesen. Während die Statistik-Angaben in der XML-Datei stets als Gesamtsummen über alle simulierten Tagen gespeichert werden, liefern diese Funktionen Werte pro Simulationtag (also z.B. nicht die Anzahl an erfolgreichen Kunden über die gesamte Simulation, sondern die mittlere Anzahl an erfolgreichen Kunden an einem Tag).

Über die Parameter der Befehle kann der Inhalt eines XML-Elements oder der Wert eines Attributes eines XML-Elements ausgelesen werden. Die Selektion eines XML-Elements erfolgt dabei mehrstufig getrennt durch "->"-Zeichen. Zwischen den "->"-Zeichen stehen jeweils de Namen von XML-Elementen. Zusätzlich können in eckigen Klammern Namen und Werte von Attributen angegeben werden, nach denen gefiltert werden soll.

Beispiele:

  • Print CallcenterModell->ModellName
    Liefert den Inhalt des Elements ModellName, welches ein Unterelement von CallcenterModell ist.
  • Print StatistikKunden[Name=Priorisierte Kunden]->StatistikKundenAnzahl[Erfolg]
    Selektiert das StatistikKunden-Element, bei dem das Name-Attribut auf den Wert Priorisierte Kunden gesetzt ist. Und liefert dann den Inhalt des Attributs Erfolg des Unterelements StatistikKundenAnzahl.
  • Print StatistikAgenten[Name=Single-Skill A]->[Anzahl]
    Selektiert das StatistikAgenten-Element, bei dem das Name-Attribut auf den Wert Single-Skill A gesetzt ist. Und liefert dann den Inhalt des Attributs Anzahl dieses Elements.

Zur Beschreibung der verfügbaren XML-Datenfelder siehe die Beschreibung der Dateiformate des Callcenter Simulators.
