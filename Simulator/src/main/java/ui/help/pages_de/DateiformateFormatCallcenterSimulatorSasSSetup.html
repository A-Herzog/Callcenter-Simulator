Konfiguration

Das Basis-Element der Konfigurationsdatei heißt <CallcenterSimulatorSasSSetup>. Es kann folgende Untereinträge besitzen:

  • Logdatei legt fest, dass die Ausgabe von Statusmeldungen nicht auf dem Bildschirm erfolgt, sondern in die angegebene Datei umgeleitet wird. Fehlt diese Angabe, so werden Meldungen auf dem Bildschirm ausgegeben.
  • Simulation: Ist dieses Element vorhanden, so wird ein Simulationsserver gestartet, auf den andere Rechner Simulationsaufgaben auslagern können. Das Simulation-Element kann optional folgende Unterelemente enthalten:
    • Port gibt an auf welchem Port der Simulationsserver seinen Dienst anbeiten soll. Vorgabewert ist Port 6783.
    • Passwort legt ein Passwort fest, mit dem die Simulationsdaten bei der Übertragung verschlüsselt werden. Wird kein Passwort angegeben, so erfolgt die Übertragung unverschlüsselt.
    • MaxThreads gibt an, wie viele Threads der Simulationsserver maximal pro Anfrage verwenden darf. Fehlt dieses Element oder wird "0" angegeben, so legt der Server die Anzahl an Threads gemäß der verfügbaren CPU-Kerne eingenständig fest.
  • Web: Ist dieses Element vorhanden, so wird ein Webserver gestartet über den auf dem Server gespeicherte Simulationsergebnisse abgerufen werden können und, wenn parallel der Simulationsserver läuft, über den ein Java-Applet abgerufen werden kann, mit dessen Hilfe direkt auf dem Simulationsserver gerechnet werden kann.
    • Port gibt an auf welchem Port der Webserver seinen Dienst anbeiten soll. Vorgabewert ist Port 80.
    • AppletJarDatei gibt den Pfad und den Dateinamen der jar-Datei an, die über den HTML-Port als GUI-Java-Applet ausgeliefert wird. Fehlt diese Angabe, versucht der Server die Applet-jar-Datei von der Hersteller-Homepage herunter zu laden.
    • StatistikVerzeichnis gibt einen Pfad zu einem Verzeichnis an, in dem sich Statistik-xml-Dateien befinden. Diese werden dann über den Webserver zugänglich gemacht. Es können beliebig viele StatistikVerzeichnis-Elemente vorhanden sein. Optional kann bei jedem Element ein Name-Attribut angegeben werden, welches angibt, wie das Verzeichnis im Webserver genannt werden soll.
    • FilterVerzeichnis gibt einen Pfad zu einem Verzeichnis an, in dem sich Filter-Skripte befinden, die über die Übersichtsseite auf einzelne Statistik-Ergebnisse angewandt werden können.

Automatische Erstellung einer Konfigurationsdatei

Über den Kommandozeilen-Befehl SaaSSystem Init kann automatisch eine Basis-Konfigurationsdatei für das SaaS-System erstellt werden.
