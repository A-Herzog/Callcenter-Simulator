File header

<!DOCTYPE CallcenterSimulatorSasSSetup SYSTEM "http://www.stochastik.tu-clausthal.de/Media/CallcenterSimulator_de.dtd">

Configuration

The root node for the configuration file is <CallcenterSimulatorSasSSetup>. It can contain the following sub nodes:

  • LogFile specifies that status messages will not be display on th screen but be written to the specified file. If this node is missing, the status messages will be written to the console output.
  • Simulation: If this element exist, a simulation server which can be used by remote computers for simulating models is started. The Simulation element can optionally contain the following sub elements:
    • Port specifies on which port the simulation server should listen for requests. The default value is port 6783.
    • Password defines a password for encrypting the simulation data. If no passwort is specfied, the data will be transfered unencrypted.
    • MaxThreads specifies the maximum number of simulation threads to be used for a single request. If this element is missing or "0" is given, the server sets up the number of threads by the number of CPU cores automatically.
  • Web: If this element exist, a web server is started. Via this web server some statistic results stored on the server can be displayed and also if the simulation server is running too, a Java applet for directly accessing the simulation server can be loaded from the web server.
    • Port specifies on which port the web server should listen for requests. The default value is port 80.
    • AppletJarFile specifies the path and the filename for the jar file which will be delivered to the client as Java applet via the HTTP port. If this node is missing the server will try to download the applet jar file from the vendors homepage.
    • StatisticFolder specifies a path to a folder containing statistic xml files. These files will be made accessable via the web server. There can be multiple StatisticFolder nodes. Additionally each node can contain a Name attribute defining how the statistic folder should be named by the web server.
    • FilterFolder specifies a path to a folder containing filter script files which can be applied on statistic results via the statistic overview page.

Automatic creation of a configuration file

By using the command line parameter SaaSSystem Init a basic configuration file for SaaS system can be created automatically.
