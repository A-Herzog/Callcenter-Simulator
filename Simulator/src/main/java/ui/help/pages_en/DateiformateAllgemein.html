Number formats

  • Individual floating point numbers are always stored with a decimal point (i.e. "123.456" or, if no fraction part is available "123"). In the simulator numbers are displayed according to locale settings, but this has no effect on the (international) storing the numbers.
  • Arrays of numbers are stored with semicolons as separators between the numbers.
  • Probabilities can be written as floating point numbers (i.e. "0.452") or as percent values (i.e. "45.2%").
  • Time data are mostly stored in the form hh:mm:ss. Some fields can contain time data with a higher precision in the form hh:mm:ss.nnn (where nnn are the milliseconds). Where which kind of time data is used is documented for each field.

Probability distributions

Data fields containing probability distributions will have the following structure:

<Name of the distribution> (<parameters>)

  • Possible values for <Name of the distribution> are: "Empirical data", "Never", "One point distribution", "Uniform distribution", "Exponential distribution", "Normal distribution", "Lognormal distribution", "Gamma distribution", "Erlang distribution", "Beta distribution", "Cauchy distribution", "Weibull distribution", "Chi^2 distribution" and "F distribution".
  • <parameters> can be one floating point number or an array of floating point numbers. In the case of an array of floating point numbers the individual numbers are separated by semicolons. The only exception is the distribution "Never" which has no parameters and therefore no bracket following the distribution name.

Specific meaning of the parameters of the individual distributions:

  • "Empirical data": The parameters contain the individual data points of the density, separated by semicolons. Are more or less data points passed than expected, the data will be scaled accordingly. The period to which the data are calculated and how many data points are expected depends on where the distribution is to be used.
  • "Never": The entire mass of the distribution is no infinity. If this value is, for example used for the waiting time tolerance distribution of clients the clients will cancel waiting not before having waiting for an unlimited long time (so will never canel waiting). After "Never" there are no bracket and not space.
  • "One point distribution": The distribution expects one parameter that indicates on which point the entire mass of the distribution shoud lie. If this distribution is specified for the holding times of an agent, serving all customers will last exactly as long as specified in the parameter.
  • "Uniform distribution": The distribution expects two parameters: the lower and the upper bound of the range of possible values.
  • "Exponential distribution": The distribution expects the mean of the exponential distribution as parameter.
  • "Normal distribution": The distribution expects two parameters: mean and standard deviation.
  • "Lognormal distribution": The distribution expects two parameters: mean and standard deviation.
  • "Gamma distribution": The distribution expects as parameters the two distribution parameters α and β.
  • "Erlang-Verteilung": The distribution expects as parameters the two distribution parameters n and λ.
  • "Beta distribution": The distribution expects four parameters: First the two shape parameters α and β followed by   the range limits of the interval, at which the distribution should take positive values.
  • "Cauchy distribution": The distribution expects two parameters: the median followed by the shape parameter.
  • "Weibull distribution": The distribution expects two parameters: the scale parameter followed by the shape parameter.
  • "Chi^2 distribution": The distribution expects as parameter the number of degrees of freedom.
  • "F distribution": The distribution expects two parameters: The number of degrees of freedom in the numerator and the number of degrees of freedom in the denominator.

Validating the XML files

The XML files used by the Callcenter Simulator can be validated by using the document definition files CallcenterSimulator_en.dtd and CallcenterSimulator_en.xsd.
