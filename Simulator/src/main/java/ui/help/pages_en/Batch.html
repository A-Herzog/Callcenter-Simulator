Batch processing

By using the function Batch processing the Callcenter Simulator can perform multiple simulation runs in immediate succession. There are two types of batch processing supported:

Process folder

In this mode a directory containing the model files to process has to be specified. The simulator will try to simulate all xml files for which there is no *_statistic.xml file as call center model files and will store the statistic results as *_statistic.xml files in the same directory.

This function is also available via the command line parameters.

Create parameter series

In this mode, the currently loaded call center model editor is simulated. But on each run of the batch simulation one model parameter is changed in a specifed way so that at the end there are statistic data of a series of simulations with a range of parameter values available. The simulation results are stored in the directory specified in the form of statistic xml output files. Please note that the results can also be can be written in a processed form in a single file for multiple simulation runs (see the dialog "Additional output settings").

Dialog "Additional output settings"

In this dialog you can set up to run some output filter right after each simulation and to append the results to a single file. By using this function a table continaing the results of the parameter series can be created easily.
